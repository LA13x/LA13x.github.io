  • 掌握交换机工作原理

  • 掌握SSH管理Cisco LAN交换机

3.1交换机简介

集线器没有mac地址表，交换机有，使用集线器时两台主机通信时网络中所有主机都可收到消息

1.交换

  • 交换机mac地址表

    • 默认： 空

    • 学习：帧中的源MAC地址和进入端口号

    • 转发：查表

      • 表中没有记录或广播帧
        • 除源端口之外的其他端口转发
      • 表中有记录
        • 转向相应端口
        • 或被过滤掉：目标MAC地址对应的端口和进入的端口是同一个端口

2.交换机转发方法

  • 交换机上数据帧转发方法
    • 存储转发交换
      • 数据缓冲 错误检查
    • 快速转发交换
      • 一旦检查到目标MAC地址就执行转发
    • 免分片交换
      • 数据帧长度达到64字节

4.分层网络设计

  • 接入层
    • 网络边缘，流量将从这里进出园区网
    • 面向用户的接入控制技术
  • 分布层
    • 隔离广播
    • 路由
    • 面向子网的网络访问策略
  • 核心层
    • 网络主干
    • 主要用途是提供错误隔离和高速主干连接

1.交换机启动顺序

  • 交换机启动顺序
    • Power-on self test
    • 运行启动加载起软件
      • 启动加载器执行低级CPU初始化

    • 启动加载器定位并加载默认IOS操作系统软件映像到内存，而且将对交换机的控制权交给IOS

3.3 安全

1.安全实践

  • 十项安全实践

    • 开发组织的书面安全策略
    • 关闭不使用的服务和端口
    • 使用强密码并经常进行更改
    • 控制对设备的物理访问
    • 使用HTTPs替换HTTP
    • 执行备份并定期测试备份的文件
    • 向员工就社会工程攻击进行培训
    • 加密并使用密码保护敏感数据
    • 实施防火墙
    • 保持软件为最新版本

2.LAN中的安全问题

  • MAC地址泛洪
    • 进入交换机的数据帧如果无法在其CAM表中找到目的MAC，交换机将通过除源端口之外的所有端口转发流量
    • 攻击者的PC运行MAC泛洪工具，最终伪造MAC地址会充满CAM表，从而使查表工作无效，交换机就像集线器一样以广播方式工作
  • 利用Telnet
    • Telnet协议是不安全的，应该用SSH代替
    • 暴力密码破解攻击
    • Telnet DoS攻击

3.交换机端口安全

  • 端口安全限制端口上所允许的有效MAC地址
  • 允许访问合法设备的MAC地址，而拒绝访问其他MAC地址
  • 任何通过未知MAC地址进行连接的其他尝试都会导致安全违规

  • 可以通过多种方式配置安全MAC地址：

    • 静态安全MAC地址 static
    • 动态安全MAC地址 Dynamic
    • 粘滞安全MAC地址 Sticky

  • 默认不启动端口安全功能

  • 手动启动后的默认配置

    • 最大安全MAC地址： 1
    • 违规模式： shutdown
    • 粘滞地址： 不启用

  • 保护未使用的端口

4.SSH

  • 配置SSH
    • 主机名称
    • 域名
    • 本地认证 或者 AAA
      • 需要用户名和密码

使用crypto key generate rsa 全局配置模式生成RSA密钥对，自动启用ssh

使用crypto key zeroize rsa 全局配置模式命令删除RSA密钥对，SSH服务器将自动禁用

ssh -l username ip-address
